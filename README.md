# IntegrationPlatformApi

Интеграционная веб-платформа, разработанная в рамках практической работы №6.  
Проект демонстрирует объединение нескольких независимых веб-сервисов в единое решение с использованием backend-логики и пользовательского интерфейса.

---

## Назначение проекта

Цель проекта — реализовать интеграционную платформу, которая:
- получает данные из внешнего сервиса напоминаний;
- обрабатывает полученную информацию;
- передаёт результаты в сервис отправки электронных писем.

Взаимодействие между сервисами реализовано по схеме:

**GET → обработка данных → POST**

---

## Используемые сервисы

В проекте используются два внешних API:

- **ReminderApi** — сервис для хранения и получения напоминаний  
  Используется для получения исходных данных (GET /api/Reminder)

- **OtpravkaApi** — сервис отправки электронных писем с отчётами  
  Используется для передачи сформированных данных (POST /api/EmailReport)

Интеграционная платформа выступает в роли промежуточного слоя между этими сервисами.

---

## Функциональные возможности

- получение данных напоминаний из ReminderApi;
- преобразование данных в формат электронного письма;
- отправка отчёта через OtpravkaApi;
- запуск интеграционного сценария через backend-метод;
- управление и проверка работы платформы через пользовательский веб-интерфейс.

---

## Пользовательский интерфейс

Для наглядной демонстрации работы платформы реализован простой веб-интерфейс (HTML + JavaScript), который позволяет:

- проверить наличие напоминаний;
- запустить интеграционный сценарий отправки отчёта;
- просмотреть HTTP-статус и результат выполнения запросов.

Интерфейс взаимодействует с backend-частью платформы и не обращается напрямую к внешним API, что позволяет избежать ограничений CORS.
